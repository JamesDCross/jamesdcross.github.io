Because UWP's are designed to run on the windows app store a sideload is used for testing. Right click on the Add-AppDevPackage file and run with the powershell to install it

How the App works

The app parses the movie file titles in the folder it is directed to using regex and other techniques. The results of the parsing are used to make calls out to the MovieDB Web API https://developers.themoviedb.org The Json received is then deserialized into instances of classes for use in C#. Databinding is then used to establish the connection between the call results and the UWP UI.

Excrept from Developer Diary

17/12/2016

I spent nearly all day working on a robust “bullet proof” regex for my movie title parsing. I realised that what I wanted was to complex for regex alone (even though I love it doing all that with one line of code). However I also found a disturbing fact that regex is less efficient than simple string methods (in some cases). This seems obvious now of course regular expressions have to be parsed, and code generated to perform the operation using string operations. But regex is still very convenient so I did the initial query of my parsing with regex and all the substring operations using C#. below a thing of beauty:

string keys1 = @"(19[0-9]\d|20[0-4]\d|2025$)";

Group 1 = title and year
Group 2 = year

So I have the index for the title and year then I use the string property that a string is a char array and send these strings off to the API. I also have the year in a separate group which could come in handy later. The file title string is also broken into these title and year groups which means if the whole group does not find a movie in the MovieDB I can move to the second group and so on. This also accounts for movies where a year is a title. There year is the best variable to use as aside from outliers the characters after the year in any title must be eliminated.
eg.

1984 2012 fgg 2013 dfgdg 2015 BluRay 720p 2012
1984 2012 fgg 2013 dfgdg 2015
1984 2012 fgg 2013
